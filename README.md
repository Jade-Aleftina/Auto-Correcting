# 英語3択 自動採点Webアプリ

スマートフォンで撮影した答案画像から、手書きの `a`、`b`、`c` を読み取り、登録済みの正答と比較して採点するWebアプリです。

既存のTesseract.js版を維持しつつ、精度改善のためにPython/FastAPI/PaddleOCRサーバーを呼び出せる構成を追加しました。OpenAI API、Google Cloud Vision、有料OCR APIは使いません。

通常はPaddleOCRサーバーを優先し、比較期間中はTesseract.jsとの結果比較もできます。

## 目的

- 既存の英語3択テストをできるだけ変更せずに使う
- マークシートを作らず、手書きの `a`、`b`、`c` を読み取る
- 無料で使える構成にする
- 1人の利用者が、スマートフォンからURLを開くだけで使えるようにする

## モジュール分離

OCR処理と採点処理は分離しています。

- `js/ocr.js`: Tesseract.jsを使い、切り出した回答欄画像から回答候補・確信度・確認要否を返します。
- `js/engines/PaddleOcrEngine.js`: FastAPI/PaddleOCRサーバーを呼び出し、Tesseract.jsと同じ形式の結果へ変換します。
- `js/grading.js`: OCR結果や手動修正後の回答を、正答・配点と照合して点数を計算します。
- `js/imageProcessor.js`: OCR前の画像補正、枠線除去、文字の中央寄せ、複数しきい値の前処理を行います。
- `js/engines/OcrEngine.js`: OCRエンジン差し替え用の共通インターフェースです。
- `js/engines/GradingEngine.js`: 採点エンジン差し替え用の共通インターフェースです。
- `server/`: PaddleOCRを実行するPython/FastAPIサーバーです。

将来AI採点へ置き換える場合は、`GradingEngine` と同じ `grade(test, answers, ocrItems)` を持つ新しいエンジンを追加し、`app.js` の生成箇所を差し替えます。OCRも同様に `OcrEngine` の `recognizeAnswerImage(canvas, options)` を実装すれば置き換えられます。

## 既存Tesseract版からの変更点

- Tesseract.jsは削除せず、引き続きブラウザ内OCRとして使えます。
- `PaddleOCRを使う`、`PaddleOCRとTesseractを比較`、`Tesseract.jsのみ` を画面で選べます。
- PaddleOCRサーバーURLと端末トークンは、利用端末のブラウザ内にだけ保存します。
- 公開コードにはサーバーURL、トークン、Cloudflareの秘密情報を含めません。
- PaddleOCRサーバー停止中は、接続確認やOCR実行時に分かりやすい日本語メッセージを表示します。

## Pythonサーバーの起動方法

PaddleOCRを使う場合だけ必要です。Tesseract.jsのみで使う場合、この手順は不要です。

1. `server/.env.example` を `server/.env` にコピーします。
2. `server/.env` の `SECRET_TOKEN` を自分だけが知る値に変更します。
3. `ALLOWED_ORIGINS` にGitHub PagesのURLを設定します。
4. PowerShellで `server/run-dev.ps1` を実行します。
5. 起動後、フロントエンドの「OCR設定」でサーバーURLと端末トークンを入力し、「接続確認」を押します。

Cloudflare Tunnelを使う場合は、Cloudflare側のトークンや認証情報をGitに入れないでください。Tunnelで発行されたHTTPS URLは、必要に応じて利用端末の設定欄へ入力してください。

## スマートフォンでの使用方法

1. URLをスマートフォンで開きます。
2. テスト設定を選びます。
3. 必要に応じて問題数、正答、配点を編集して保存します。
4. 「カメラで撮影」または「撮影済み画像を選択」から答案画像を読み込みます。
5. 画像の向き、明るさ、コントラスト、二値化を調整します。
6. 各列の一番上と一番下の回答文字をタップして、回答欄位置を自動配置します。
7. OCRを開始します。
8. 確認画面で、各問題の切り出し画像、OCR結果、確信度、正答、正誤を確認します。
9. 不確かな問題は a/b/c/未回答 ボタンで修正します。
10. 確定して保存します。

## テスト登録方法

「テスト設定」画面で次を登録します。

- テスト名
- 問題数
- 各問題の正答 `a`、`b`、`c`
- 各問題の配点
- 答案画像を端末内に保存するか

初期状態では20問のサンプルテストが作成されます。問題数は後から変更できます。

## 回答範囲の登録方法

1. 答案画像を読み込みます。
2. 「回答範囲の指定」画面へ進みます。
3. 通常は「上下だけ指定」を選びます。
4. 開始問題、問題数、列数、読み順を実際の答案用紙に合わせます。
5. 画面の案内に従って、各列の一番上と一番下の回答文字の中心をタップします。
6. 24問が1列なら2タップ、2列なら4タップで、間の問題は自動配置されます。
7. ずれている問題は、一覧から「修正」を押して中心をタップし直します。
8. 切り出し範囲が大きすぎる、または枠線が入る場合は「切り出し幅」「切り出し高さ」を小さくします。
9. 「回答範囲を保存」を押します。

回答欄の座標は画像全体に対する比率で保存します。そのため、次回以降に画像サイズが変わっても、おおむね同じ位置を切り出せます。

### 上下だけ指定がおすすめです

スマートフォン写真では、枠線が少し歪む、影が入る、用紙が斜めになる、線が途切れる、ということがよくあります。そのため、枠線を完全に自動検出するよりも、最初に一度だけ回答文字の中心を登録するほうが安定しやすいです。

上下だけ指定では、列ごとに一番上と一番下の回答文字の中心だけを指定します。アプリはその2点の間にある問題を等間隔に配置します。回答欄全体ではなく `a`、`b`、`c` の文字が書かれる場所だけを小さく切り出すため、枠線や問題番号がOCR画像に入りにくくなります。

中心タップ方式は、上下だけ指定でずれた問題を個別に直すための補助として使います。

### 枠線から自動検出

枠線がはっきり写っている回答用紙では、「枠線から自動検出」も使えます。ただし、これは補助機能です。スマートフォン写真では上下だけ指定のほうが安定することがあります。

1. 答案画像を読み込み、「回答範囲の指定」画面へ進みます。
2. 開始問題、問題数、列数、読み順、余白を設定します。
3. まず「範囲をまとめて」で、回答が並んでいる大枠を大きめに囲みます。
4. 「大枠内を自動検出」を押します。
5. 検出された回答欄が画像上に表示されます。
6. ずれている問題だけ、一覧から「修正」を押して中心をタップし直します。
7. 「回答範囲を保存」を押します。

画像全体から探したい場合は「枠線から自動検出」も使えますが、問題文や表の線まで拾いやすいため、使う場合は「大枠内を自動検出」から試してください。

旧手順:

1. 答案画像を読み込み、「回答範囲の指定」画面へ進みます。
2. 開始問題、問題数、列数、読み順、余白を設定します。
3. 「枠線から自動検出」を押します。
4. 検出された回答欄が画像上に表示されます。
5. ずれている問題だけ、一覧から「修正」を押して個別に囲み直します。
6. 「回答範囲を保存」を押します。

自動検出は、画像内の水平線・垂直線を探し、その交差から四角い回答欄候補を作ります。写真の傾きや遠近が大きい場合、影が強い場合、枠線が途切れている場合は検出がずれることがあります。その場合は、画像調整の明るさ・コントラスト・二値化を変えて再試行するか、個別修正で上書きしてください。

### 範囲をまとめて指定

「範囲をまとめて指定」では、回答が書かれている範囲を1回だけ囲みます。

たとえば20問が1列に並んでいる場合:

- 開始: 1
- 問題数: 20
- 列数: 1
- 読み順: 上から下、次の列

20問が2列に分かれていて、左列が1〜10、右列が11〜20の場合:

- 開始: 1
- 問題数: 20
- 列数: 2
- 読み順: 上から下、次の列

1行に複数問が並ぶ形式の場合は、読み順を「左から右、次の行」にします。

余白は、自動分割した各セルの内側を少し狭める設定です。問題番号や枠線がOCRに混ざる場合は、余白を少し大きくしてください。

## 採点方法

OCR後、確認画面に次が表示されます。

- 問題番号
- 切り出した回答画像
- OCRが読み取った回答
- OCRの確信度
- 正答
- 正解、不正解、未回答
- a、b、c、未回答の手動修正ボタン

確信度が低い問題、未回答、複数文字、a/b/c以外の読み取りは上部に表示されます。保存前に必ず確認してください。

## CSV出力方法

「保存・出力」画面で次を選べます。

- `今回分CSV`: 直近で保存した1回分だけ出力します。
- `条件CSV`: 期間やテスト名で絞り込んだ結果を一括出力します。

CSVは日本語版Excelで文字化けしにくいUTF-8 BOM付きです。問題ごとの最終回答、OCR回答、確信度、点数も列として出力します。

## バックアップと復元

採点結果、テスト設定、回答欄位置はブラウザ内の IndexedDB に保存されます。

- `JSONバックアップ`: 全データを書き出します。
- `JSON復元`: 書き出したJSONから復元します。

重要:

- 利用者のデータは各端末のブラウザ内に保存されます。
- ブラウザデータを削除すると、採点結果や設定は消えます。
- 定期的にCSVまたはJSONでバックアップしてください。

## Tesseract.jsの制約

Tesseract.jsは無料で使えるOCRですが、手書き文字の認識は誤認識が起こり得ます。

特に次のケースでは誤認識しやすくなります。

- 文字が薄い
- 写真が暗い
- 影がある
- ピントが合っていない
- `a` と `c` が似ている
- `b` が崩れている
- 回答欄から文字がはみ出している
- 枠線や罫線が回答文字と近い

このアプリはOCR結果をそのまま確定せず、確信度が低い問題を確認対象にします。最終保存前に必ず目視確認してください。

### 枠線対策

現在の版では、OCR前に次の補正を自動で行います。

- 回答欄の外周にある枠線を白く消す
- 長い縦線・横線を検出して消す
- 小さなゴミを除去する
- 残った文字だけを中央に拡大する
- しきい値を変えた複数の補正画像でOCRし、一番よい候補を採用する

確認画面には、元の切り出し画像に加えてOCR用の補正画像も表示します。補正画像で文字が消えすぎている場合は、画像調整画面で二値化やコントラストを弱めて再実行してください。

## 精度を上げる撮影方法

アプリ内にも同じ案内を表示しています。

- 答案全体を明るい場所で撮影する
- 影が入らないようにする
- 紙に対してできるだけ正面から撮影する
- 回答文字を濃く書く
- a、b、cは小さすぎないように書く
- 回答欄から文字がはみ出さないようにする
- 写真がぼやけている場合は撮り直す

## 現在の制約

- 用紙の外枠検出、特徴点検出、遠近補正は未実装です。
- 四隅を手動指定して射影補正する機能は未実装です。
- 回答範囲を自動分割できますが、行間や列幅が大きく不均一な答案では個別修正が必要です。
- 精度検証専用画面は今後追加予定です。現時点では `tests/manual-test-plan.md` に沿って手動検証してください。
- Tesseract.js本体はCDNから読み込みます。OCR対象の答案画像は外部OCRサービスへ送信しませんが、初回表示時にライブラリ取得のためインターネット接続が必要です。

## 検証

最低限の確認項目は `tests/manual-test-plan.md` にまとめています。

実機確認では、少なくとも次を確認してください。

- aを正しく認識できる
- bを正しく認識できる
- cを正しく認識できる
- A、B、Cを小文字へ変換できる
- 空欄を未回答と判定できる
- 不明な文字を自動確定しない
- 利用者が回答を修正できる
- 正答との比較が正しい
- CSVが正常に開ける
- スマートフォン画面で操作できる
- 答案画像が外部OCRサービスへ送信されていない
