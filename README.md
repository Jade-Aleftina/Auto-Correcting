# 英語3択 自動採点Webアプリ

スマートフォンで撮影した答案画像から、手書きの `a`、`b`、`c` を Tesseract.js で読み取り、登録済みの正答と比較して採点する静的Webアプリです。

OpenAI API、Google Cloud Vision、有料OCR API、APIキーは使いません。画像処理とOCRは利用者のブラウザ内で行い、答案画像を外部OCRサービスへ送信しません。

## 目的

- 既存の英語3択テストをできるだけ変更せずに使う
- マークシートを作らず、手書きの `a`、`b`、`c` を読み取る
- 無料で使える構成にする
- 1人の利用者が、スマートフォンからURLを開くだけで使えるようにする

## モジュール分離

OCR処理と採点処理は分離しています。

- `js/ocr.js`: Tesseract.jsを使い、切り出した回答欄画像から回答候補・確信度・確認要否を返します。
- `js/grading.js`: OCR結果や手動修正後の回答を、正答・配点と照合して点数を計算します。
- `js/engines/OcrEngine.js`: OCRエンジン差し替え用の共通インターフェースです。
- `js/engines/GradingEngine.js`: 採点エンジン差し替え用の共通インターフェースです。

将来AI採点へ置き換える場合は、`GradingEngine` と同じ `grade(test, answers, ocrItems)` を持つ新しいエンジンを追加し、`app.js` の生成箇所を差し替えます。OCRも同様に `OcrEngine` の `recognizeAnswerImage(canvas, options)` を実装すれば置き換えられます。

## 起動方法

### もっとも簡単な確認方法

このフォルダをGitHub Pagesなどの静的ホスティングに置き、`index.html` を開きます。

### ローカルで確認する場合

ES Modulesを使用しているため、ブラウザでHTMLファイルを直接開くのではなく、簡易Webサーバー経由で開いてください。

Pythonが使える場合:

```bash
python -m http.server 8000
```

その後、ブラウザで次を開きます。

```text
http://localhost:8000/
```

## スマートフォンでの使用方法

1. URLをスマートフォンで開きます。
2. テスト設定を選びます。
3. 必要に応じて問題数、正答、配点を編集して保存します。
4. 答案画像を撮影、または端末内の画像を選択します。
5. 画像の向き、明るさ、コントラスト、二値化を調整します。
6. 回答欄の位置を画像上で四角く囲み、保存します。
7. OCRを開始します。
8. 確認画面で、各問題の切り出し画像、OCR結果、確信度、正答、正誤を確認します。
9. 不確かな問題は a/b/c/未回答 ボタンで修正します。
10. 確定して保存します。

## テスト登録方法

「テスト設定」画面で次を登録します。

- テスト名
- 問題数
- 各問題の正答 `a`、`b`、`c`
- 各問題の配点
- 答案画像を端末内に保存するか

初期状態では20問のサンプルテストが作成されます。問題数は後から変更できます。

## 回答欄位置の登録方法

1. 答案画像を読み込みます。
2. 「回答欄の指定」画面へ進みます。
3. 問1から順に、回答欄をドラッグして四角で囲みます。
4. 位置を間違えた場合は、該当する問題の「削除」を押して指定し直します。
5. 「回答欄位置を保存」を押します。

回答欄の座標は画像全体に対する比率で保存します。そのため、次回以降に画像サイズが変わっても、おおむね同じ位置を切り出せます。

## 採点方法

OCR後、確認画面に次が表示されます。

- 問題番号
- 切り出した回答画像
- OCRが読み取った回答
- OCRの確信度
- 正答
- 正解、不正解、未回答
- a、b、c、未回答の手動修正ボタン

確信度が低い問題、未回答、複数文字、a/b/c以外の読み取りは上部に表示されます。保存前に必ず確認してください。

## CSV出力方法

「保存・出力」画面で次を選べます。

- `今回分CSV`: 直近で保存した1回分だけ出力します。
- `条件CSV`: 期間やテスト名で絞り込んだ結果を一括出力します。

CSVは日本語版Excelで文字化けしにくいUTF-8 BOM付きです。問題ごとの最終回答、OCR回答、確信度、点数も列として出力します。

## バックアップと復元

採点結果、テスト設定、回答欄位置はブラウザ内の IndexedDB に保存されます。

- `JSONバックアップ`: 全データを書き出します。
- `JSON復元`: 書き出したJSONから復元します。

重要:

- 利用者のデータは各端末のブラウザ内に保存されます。
- ブラウザデータを削除すると、採点結果や設定は消えます。
- 定期的にCSVまたはJSONでバックアップしてください。

## Tesseract.jsの制約

Tesseract.jsは無料で使えるOCRですが、手書き文字の認識は誤認識が起こり得ます。

特に次のケースでは誤認識しやすくなります。

- 文字が薄い
- 写真が暗い
- 影がある
- ピントが合っていない
- `a` と `c` が似ている
- `b` が崩れている
- 回答欄から文字がはみ出している

このアプリはOCR結果をそのまま確定せず、確信度が低い問題を確認対象にします。最終保存前に必ず目視確認してください。

## 精度を上げる撮影方法

アプリ内にも同じ案内を表示しています。

- 答案全体を明るい場所で撮影する
- 影が入らないようにする
- 紙に対してできるだけ正面から撮影する
- 回答文字を濃く書く
- a、b、cは小さすぎないように書く
- 回答欄から文字がはみ出さないようにする
- 写真がぼやけている場合は撮り直す

## 現在の制約

- 用紙の外枠検出、特徴点検出、遠近補正は未実装です。
- 四隅を手動指定して射影補正する機能は未実装です。
- 不要部分の切り取りは、回答欄ごとの切り出しとして実装しています。答案全体の自由トリミングは未実装です。
- 精度検証専用画面は今後追加予定です。現時点では `tests/manual-test-plan.md` に沿って手動検証してください。
- Tesseract.js本体はCDNから読み込みます。OCR対象の答案画像は外部OCRサービスへ送信しませんが、初回表示時にライブラリ取得のためインターネット接続が必要です。

## 検証

最低限の確認項目は `tests/manual-test-plan.md` にまとめています。

実機確認では、少なくとも次を確認してください。

- aを正しく認識できる
- bを正しく認識できる
- cを正しく認識できる
- A、B、Cを小文字へ変換できる
- 空欄を未回答と判定できる
- 不明な文字を自動確定しない
- 利用者が回答を修正できる
- 正答との比較が正しい
- CSVが正常に開ける
- スマートフォン画面で操作できる
- 答案画像が外部OCRサービスへ送信されていない
